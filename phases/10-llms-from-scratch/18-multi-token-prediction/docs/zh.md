# 多 Token 预测（MTP）

> 从 GPT-2 到 Llama 3，每个自回归 LLM 都在每个位置上训练一个损失：预测下一个 token。DeepSeek-V3 在每个位置上增加了第二个损失：预测再下一个 token。额外的 14B 参数（在 671B 模型上）通过梯度流蒸馏回主模型，而训练好的 MTP 头在推理时被重新用作推测解码（speculative decoding）的草稿模型，接受率达到 80%+。1.8 倍的生成吞吐量免费获得。本课从 DeepSeek 技术报告构建逐层的 MTP 模块，计算损失和共享头参数布局，并解释为什么 MTP 保持因果链而 Gloeckle 等人的原始并行 MTP 破坏了它。

**Type:** Build
**Languages:** Python (stdlib)
**Prerequisites:** Phase 10 · 04（预训练一个迷你 GPT）、Phase 10 · 15（推测解码）
**Time:** ~60 分钟

## Learning Objectives

- 陈述 MTP 训练目标并推导跨预测深度的联合损失。

- 计算在预训练运行中添加 MTP 模块的参数和内存开销。
- 从零开始实现一个 MTP 模块：共享 embedding、每深度的 Transformer 块、投影和共享输出头。

## 问题

下一个 token 预测是标准的 LLM 训练目标。每个隐藏状态被监督来预测唯一一件事：紧接着的下一个 token。这是一个令人惊讶地弱的信号。序列中的大部分信息超越一个 token——结构、连贯性、事实性、算术流程。模型必须通过数万亿个 token 累积许多单 token 信号来学习这些。

MTP 提出问题：如果每个隐藏状态被监督来同时预测多个未来 token 会怎样？Gloeckle 等人（Meta，2024）表明这有帮助。他们的实现在骨干网络之上放置了几个独立的输出头，每个预测不同的偏移量。并行的，简单的，但这些头看到的是同一个隐藏状态，没有任何层次化的精炼——并且预测不在因果上链接，因此无法用于推测解码。

DeepSeek-V3（2024 年 12 月）将 MTP 重新设计为逐层模块，在每个预测深度上保持因果链。模型从 `h_i^(0)` 预测 `t+1`，然后从组合了 `h_i^(0)` 和 `E(t+1)` embedding 的新隐藏状态 `h_i^(1)` 预测 `t+2`，依此类推。每个深度是其自己的小型 Transformer 块。共享 embedding 和共享输出头使参数开销保持适中。在 DeepSeek-V3 的规模上，MTP 模块跨多个深度共 14B 额外参数，叠加在 671B 主模型权重之上。这 2% 的开销换来了更密集的训练信号以及推理时现成的推测解码草稿模型。

本课从零开始构建一个 MTP 模块和 D 深度损失。数学很整洁。实现约 150 行。

## 概念

### 逐层 MTP 配方

DeepSeek-V3 在主模型之上添加 `D` 个 MTP 模块。每个模块 `k`（`k = 1..D`）预测深度 `k` 处的 token——即给定前缀至位置 `i`，预测 `t_{i+k}`。

模块 `k` 包含：

- 一个带有自己的注意力和 MLP 的 Transformer 块 `T_k`。
- 一个投影矩阵 `M_k`，将前一层深度的隐藏状态与下一深度真实 token 的 embedding 组合。
- 共享 embedding `E`（与主模型相同）。
- 共享输出头 `Out`（与主模型相同）。

在训练时，对于前缀至位置 `i`，每深度的隐藏状态是：

```
h_i^(0) = 主模型骨干在位置 i 的输出
h_i^(k) = T_k( M_k * concat(RMSNorm(h_i^(k-1)), RMSNorm(E(t_{i+k}))) )   对于 k >= 1
```

每深度的预测是：

```
logits_{i+k} = Out(h_i^(k-1))   对于 k = 1..D
```

每深度的损失是对真实 `t_{i+k}` 的交叉熵（cross-entropy）：

```
L_k = CE(logits_{i+k}, t_{i+k})
```

跨深度的联合损失：

```
L_MTP = (lambda / D) * sum_{k=1..D} L_k
```

`lambda` 是一个小的权重因子——DeepSeek-V3 在训练的前 10% 使用 0.3，之后使用 0.1。总训练损失是 `L_main + L_MTP`。

### 为什么是逐层，而非并行

Gloeckle 的原始并行 MTP 有 D 个输出头，每个直接应用于 `h_i^(0)`。每个头从相同的骨干隐藏状态预测 `t_{i+k}`。这训练起来没问题，但预测不互为条件。你不能使用 `head_1` 的输出来帮助 `head_2`——这些头并行触发。

DeepSeek-V3 的逐层设计从 `h_i^(k-1)` 加上实际的下一个 token embedding `E(t_{i+k})` 构建 `h_i^(k)`。这保持了因果链：要预测 `t_{i+k+1}`，深度 `k+1` 处的模块可以看到 `t_{i+k}` 是什么。这在结构上与自回归 decoder 消费自身输出的方式相同——使得 MTP 模块可以直接用作推测解码的草稿模型。

在推理时：将 `h_i^(k-1)` 和草拟的 `t_{i+k}` 输入模块 `k+1`，获得对 `t_{i+k+1}` 的预测。重复。这正是 EAGLE 风格的草稿，使用训练好的 MTP 模块作为草稿网络。DeepSeek-V3 报告第一个 MTP 模块的接受率达到 80%+，加速约 1.8 倍。

### 参数核算

对于一个隐藏维度为 `h`、词汇量为 `V` 的模型：

- 主模型：数十亿参数，加上一个大小为 `V * h` 的输出头。
- 共享输出头：复用主模型的头。无额外参数。
- 共享 embedding：复用主模型的 embedding。无额外参数。
- 每个 MTP 模块：
  - 投影 `M_k`：`(2h) * h = 2h^2`。
  - Transformer 块 `T_k`：注意力（MHA 为 `4h^2`）加上 MLP（对于比例为 8/3 的 SwiGLU 通常为 `8h^2`）。约每个块 `12h^2`。

每个模块的总额外参数：`~14h^2`。对于 DeepSeek-V3 的 `h = 7168`，D = 1 个模块：论文中 `~14 * 7168^2 = ~720M` 参数。DeepSeek-V3 报告为 14B——差异主要在于 MTP 模块中的专家层也是 MoE 结构。

### 推测解码的回报

在预训练期间，MTP 模块使训练减慢约 10%（更多的前向计算、额外的损失）。回报是双重的：

1. 更密集的训练信号。每个隐藏状态看到 D+1 个监督目标。在 DeepSeek-V3 的消融实验中，对 MMLU、GSM8K、MATH、HumanEval 的测量效果：持续数个百分点的提升。

2. 推理时免费的推测解码草稿。MTP 模块已经被训练来预测接下来的几个 token。重新用作草稿网络时，它带来 80%+ 的接受率。在此水平上，N=3 或 N=5 的推测解码可提供 1.8 倍的吞吐量。10% 的训练时间代价在第一次运行推理时就已经回本。

### 与 EAGLE 的关系

EAGLE 在预训练之后单独训练一个小型草稿模型。MTP 将草稿嵌入到预训练中。两种方法收敛到相似的接受率，但通过不同的流水线：

| 维度 | EAGLE-3 | MTP (DeepSeek-V3) |
|-----------|---------|------------------|
| 训练时机 | 预训练后 | 预训练期间 |
| 与现有权重的向后兼容 | 是 | 否（需要重新训练） |
| 草稿参数 | 1-2 个 Transformer 层 | 1 个 Transformer 块 + 投影 |
| 接受率 | 0.88-0.92 | 深度 1 时 0.80+ |
| 超越加速的益处 | 仅推测解码 | 更密集的训练信号 + 加速 |

## 构建它

`code/main.py` 端到端构建一个 MTP 模块：共享 embedding、投影、Transformer 块、共享输出头。然后计算一个短合成序列上的每深度交叉熵损失，并按组件打印参数数量。使用 32 个 token 的玩具词汇量使数字保持可读。

### 步骤 1：共享 embedding 表

一个单一的 `vocab_size x hidden` 表被主模型和每个深度上的每个 MTP 模块使用。不是第二个副本——字面上是同一个张量。

### 步骤 2：每深度的组合

```python
def combine(prev_hidden, next_token_embed, M_k):
    # 沿特征维度拼接，然后投影到 hidden
    concat = rms_norm(prev_hidden) + rms_norm(next_token_embed)  # 向量加法的替身
    projected = matvec(M_k, concat)
    return projected
```

真实的 DeepSeek-V3 将两个 RMSNorm 后的向量拼接为 `[2h]`，并使用 `h x 2h` 矩阵投影。玩具版使用向量加法以保持标准库简洁。

### 步骤 3：深度 k 处的 Transformer 块

自注意力加 MLP。在玩具版中，一个单层线性注意力块和一个 SwiGLU MLP 使结构可见，无需 numpy。

### 步骤 4：共享输出头

复用主模型的输出投影。词汇表上的 logits。

### 步骤 5：每深度的损失

softmax(logits) 相对于偏移量 `k` 处真实 token 的交叉熵。使用 `lambda / D` 缩放因子跨深度聚合。

### 步骤 6：参数核算

打印总参数计数、共享（embedding、输出头）计数以及每个模块的额外计数。展示 MTP 额外部分与主模型大小的比例。

## 使用它

MTP 已集成到 DeepSeek-V3（2024 年 12 月）和 DeepSeek-R1 系列中。在推理时：

- DeepSeek 自己的推理栈将 MTP 模块作为推测解码器直接开箱使用。
- 截至 2026 年 4 月，vLLM 和 SGLang 已有 DeepSeek-V3 MTP 的集成路径。
- AMD 的 ROCm SGLang 教程展示了一个具体的 MTP 推测解码配置，在 V3 检查点上实测达到 1.8 倍加速。

何时在新的预训练运行中使用 MTP：

- 你控制完整的预训练流水线，并希望存入更密集的训练信号。
- 你知道你将大规模部署模型，并希望免费获得推测解码。
- 你的隐藏大小至少为 4096。在 1B 规模时，开销比收益更伤人。

何时不使用：

- 微调现有的预训练稠密模型。MTP 模块未经过训练。
- 你需要一个干净的基线来对比较的研究模型。MTP 改变了架构。

## 练习

1. 运行 `code/main.py`。展示随合成信号增强，每深度的损失单调递减。修改合成数据使用固定模式，验证深度 1 和深度 2 的损失均收敛。

2. 对于一个稠密 70B 模型（hidden 8192，80 层），使用 D=1 MTP 模块计算参数开销。与 DeepSeek-V3 报告的 14B 开销进行比较。解释为什么 DeepSeek 的数字更高：MTP Transformer 块继承了相同的 MoE 结构，膨胀了每个模块的参数数量。

3. 在玩具中实现 D=2：添加第二个 MTP 模块，接收 h^(1) 并预测 `t_{i+2}`。验证联合损失和参数核算与 DeepSeek 论文的方程 19-21 一致。

4. 将玩具切换到并行 MTP（Gloeckle 风格）：在主隐藏状态之上添加 D 个输出头，每个预测不同的偏移量。测量在相同合成信号上每深度损失与逐层版本的比较。对于 k > 1，逐层版本应产生更低的深度 k 损失，因为它以前面层级的预测为条件。

5. 使用训练好的 MTP 模块作为 EAGLE 风格的草稿：在推理时调用模块 k 来提议 `t_{i+k}`。测量这些草稿 token 在一段保留序列上相对于主模型预测的接受率。如果在玩具上达到 50%+，你就复现了 MTP 作为草稿的经验属性。

## 关键术语

| 术语 | 人们常说的 | 实际含义 |
|------|----------------|------------------------|
| MTP 模块 | "额外的损失块" | 一个小型 Transformer 块加投影，预测主模型前方 k 个位置的 token |
| 预测深度 | "哪个偏移量" | 整数 k，使得模块 k 从前缀至位置 i 预测 `t_{i+k}` |
| 并行 MTP | "Gloeckle 风格" | 同一骨干隐藏状态上的 D 个独立头，无条件链 |
| 逐层 MTP | "DeepSeek-V3 风格" | 每个模块以前一层深度的隐藏状态加下一 token 的 embedding 为条件；保持因果链 |
| 共享输出头 | "复用主头" | MTP 模块调用主模型的 LM 头，而非单独的输出投影 |
| 共享 embedding | "复用主表" | 同一词汇 embedding 表各处通用；无重复参数 |
| 投影矩阵 M_k | "组合隐藏状态和下一 token" | 一个 `h x 2h` 的线性层，将前一层隐藏状态和目标 token embedding 折叠为下一深度的输入 |
| 联合损失 L_MTP | "平均额外损失" | 每深度交叉熵损失的算术平均，由 `lambda` 缩放 |
| 深度 1 处的接受率 | "MTP 草稿有多经常正确" | D=1 MTP 模块的 top-1 预测等于主模型 top-1 预测的频率；DeepSeek-V3 上为 80%+ |
| Lambda 权重 | "额外损失的重要性" | 每深度的缩放因子；DeepSeek-V3 在训练开始时为 0.3，后期为 0.1 |
