# 多模态代理与计算机使用（终章）

> 2026 年前沿产品是一个多模态代理，读取截图、点击按钮、导航网页 UI、填写表单、端到端完成工作流。SeeClick 和 CogAgent（2024）证明了 GUI 定位原语。Ferret-UI 增加了移动端。ChartAgent 为图表引入了视觉工具使用。VisualWebArena 和 AgentVista（2026）是前沿追逐的基准——而 Gemini 3 Pro 和 Claude Opus 4.7 在 AgentVista 的困难任务上得分仅约 30%。这个终章将 Phase 12 的每一条线索汇聚在一起：感知（高分辨率 VLM）、推理（带工具使用的 LLM）、定位（坐标输出）、长时域记忆和评估。

**Type:** Capstone
**Languages:** Python (stdlib, action schema + agent loop skeleton)
**Prerequisites:** Phase 12 · 05 (LLaVA), Phase 12 · 09 (Qwen-VL JSON), Phase 14 (Agent Engineering)
**Time:** ~240 minutes

## 学习目标

- 设计多模态代理循环：感知 → 推理 → 行动 → 观察 → 重复。
- 构建 VLM 可以以 JSON 形式发出的 GUI 定位输出 schema（点击坐标、输入文本、滚动、拖拽）。
- 比较纯截图代理 vs 无障碍树代理 vs 混合代理。
- 在一个小型 VisualWebArena 切片上设置多模态代理基准评估。

## 问题

一个订票网站工作流："给我找一张 4月15日去东京的航班，过道座位，800美元以下，订它。"

多模态代理需要：

1. 截取浏览器截图。
2. 将截图 + URL + 目标解析为计划。
3. 发出结构化动作：点击（在 x,y 处）、输入 "Tokyo"（在元素 E 处）、向下滚动、选择（单选按钮）。
4. 将动作应用到浏览器。
5. 观察新状态（下一张截图）。
6. 重复直到任务完成。

每一步都是一个多模态 VLM 调用。VLM 输出必须是可解析的 JSON。错误跨步骤累积，所以恢复很重要。

## 概念

### GUI 定位——原语

GUI 定位是：给定截图和自然语言指令，输出要点击的 (x, y) 坐标（或其他动作）。

SeeClick（arXiv:2401.10935）是第一个规模化开源结果：在合成+真实 GUI 数据上微调 VLM，将坐标作为纯文本 token 输出。有效。

CogAgent（arXiv:2312.08914）为密集 UI 增加了 1120x1120 高分辨率编码。分数：网页导航约 84%。

Ferret-UI（arXiv:2404.05719）专注于移动 UI，与 iOS 无障碍数据集成。

输出格式通常是 JSON：

```json
{"action": "click", "x": 384, "y": 220, "element_desc": "Search button"}
```

`element_desc` 有助于恢复：如果坐标在截图之间漂移，语义提示让系统可以重新定位。

### 动作 Schema

典型动作 schema 有 6-10 种动作类型：

- `click`：(x, y)
- `type`：(text, x?, y?)
- `scroll`：(direction, amount)
- `drag`：(x0, y0, x1, y1)
- `select`：(option_index)
- `hover`：(x, y)
- `navigate`：(url)
- `wait`：(ms)
- `done`：(success, explanation)

代理每步发出一个动作。浏览器包装器执行并返回新状态。

### 纯截图 vs 无障碍树

两种输入模式：

- 纯截图：完整图像，无结构信息。最通用；适用于任何应用。
- 无障碍树：结构化 DOM / iOS 无障碍信息。定位可靠得多；在树可用处有效。
- 混合：两者，树作为原子动作的可靠定位器，截图为语义上下文。

生产代理在可能时使用混合。浏览器自动化（Selenium + 无障碍）始终有树；桌面应用有时有。

### 长时域记忆

20 步工作流生成 20 张截图。VLM 上下文填充得很快。三种压缩策略：

- 摘要链：每 5 步后总结已发生的事情，丢弃旧截图。
- 跳帧：保留第一张、最后一张和每第 3 张截图。
- 工具记录日志：执行动作，保留已做事情的文本日志；不重新查看旧截图。

Claude 的 computer-use API 使用日志模式。更简单、更可靠。

### 视觉工具使用

ChartAgent（arXiv:2510.04514）为图表理解引入了视觉工具使用：裁剪、缩放、OCR、调用外部检测。代理可以输出 "crop to region (100, 200, 300, 400) then call OCR" 作为工具调用。工具返回文本；VLM 继续推理。

这个模式是通用的：标记集合提示、区域注释和外部检测工具都适合相同的"输出工具调用，接收结构化响应"schema。

### 2026 年基准

- ScreenSpot-Pro。约 1000 张网页截图的 GUI 定位。开源 SOTA Qwen2.5-VL-72B 约 85%。前沿约 90%。
- VisualWebArena。端到端网页任务（购物、论坛、分类信息）。开源 SOTA 约 20%。Gemini 3 Pro 约 27%。
- AgentVista（arXiv:2602.23166）。2026 年最难的基准。跨 12 个领域的现实工作流。前沿模型得分 27-40%；开源模型 10-20%。
- WebArena / WebShop。旧基准；被前沿饱和。

### 为什么仍然困难

代理性能瓶颈：

1. 细尺度视觉定位。"点击小 X"在移动体验分辨率下经常失败。
2. 长时域规划。10 个动作后，代理偏离目标。
3. 错误恢复。当点击失败（错误按钮），检测+恢复很少在训练数据中。
4. 跨页上下文。在标签页之间跳转或长表单丢失状态。

研究方向：记忆架构、显式重新规划、多模态验证（行为成功的截图匹配）。

### 终章构建

终章任务：构建一个计算机使用代理，它：

1. 阅读订票网站模拟页面的 HTML + 截图。
2. 规划多步序列：搜索 → 选择 → 填写表单 → 提交。
3. 发出匹配动作 schema 的 JSON 动作。
4. 在固定的 10 任务切片上评估。

本课提供易于扩展到真实浏览器的脚手架代码。

## 使用它

`code/main.py` 是终章脚手架：

- 动作 schema JSON 定义（10 个动作）。
- 模拟浏览器状态为 dict。
- 代理循环骨架：接收状态、发出动作、应用、循环。
- 10 任务小型基准（合成页面）衡量端到端成功率。
- 动作失败时的错误恢复钩子。

本课产出 `outputs/skill-multimodal-agent-designer.md`。给定计算机使用产品（领域、动作集、评估目标），设计完整代理循环、记忆策略、定位模式和预期基准分数。

## 练习

1. 用 `screenshot_region` 工具（裁剪 + 缩放）扩展动作 schema。什么任务受益？

2. 阅读 AgentVista（arXiv:2602.23166）。描述最困难的任务类别以及为什么前沿模型仍然失败。

3. 长时域记忆压缩：设计一个摘要链，保留最多 4 张活跃截图，任意数量记录在案。

4. 构建一个错误恢复钩子：动作失败（按钮未找到）时，代理接下来做什么？

5. 在 10 个网页任务上比较纯截图 Claude 4.7 和混合截图 + 无障碍树 Qwen2.5-VL。哪个在哪些任务上胜出？

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|---------|
| GUI 定位 | "点击坐标" | 模型为截图上指令的目标输出 (x,y) |
| 动作 schema | "工具定义" | 有效动作的 JSON 描述（点击、输入、滚动、拖拽） |
| 无障碍树 | "结构化 DOM" | 来自浏览器/iOS API 的机器可读 UI 层级 |
| 混合代理 | "截图 + 树" | 同时使用图像和结构化信息；比任一单独使用更可靠 |
| 视觉工具使用 | "缩放/裁剪/检测" | 代理在计划中调用外部视觉工具（OCR、检测） |
| 摘要链 | "记忆压缩" | 定期文本摘要替代长截图历史 |
| VisualWebArena | "端到端网页基准" | 2024 年端到端网页任务基准 |
| AgentVista | "2026 年硬基准" | 12 领域现实工作流；即使 Gemini 3 Pro 也得分约 30% |
