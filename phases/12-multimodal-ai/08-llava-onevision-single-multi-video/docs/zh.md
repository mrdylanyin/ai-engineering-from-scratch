# LLaVA-OneVision：单图像、多图像、视频合一模型

> 在 LLaVA-OneVision（Li 等人, 2024年8月）之前，开源 VLM 世界有不同的谱系：LLaVA-1.5 用于单图像，Mantis 和 VILA 用于多图像模型，Video-LLaVA 和 Video-LLaMA 等视频模型。每个在自己基准上赢，在其他基准上输。LLaVA-OneVision 论证了一个课程就可以训练一个模型在所有三种场景中取胜，而且涌现出的任务迁移效果（单图像技能输出到视频，多图像推理输出到单图像）优于专家的总和。配方出奇简单：一个跨场景保持恒定的视觉 token 预算，加上一个从单图像到 OneVision（多图像）再到视频的显式课程。本课阅读预算、课程和涌现行为。

**Type:** Build
**Languages:** Python (stdlib, token budget solver + curriculum planner)
**Prerequisites:** Phase 12 · 05 (LLaVA), Phase 12 · 06 (any-resolution)
**Time:** ~180 minutes

## 学习目标

- 设计一个跨单图像、多图像和视频输入保持恒定的视觉 token 预算。
- 编排一个将技能从单图像迁移到视频而不会灾难性遗忘的训练课程。
- 解释为什么当课程做得正确时，单一模型在相同参数数量下优于专家模型。
- 说出 LLaVA-OneVision 报告的三个涌现能力：多摄像头推理、标记集合提示、iPhone 截图智能体。

## 问题

图像、多图像和视频对模型施加不同的压力。

单图像需要高分辨率 token（AnyRes，约 2880 个视觉 token）来捕捉 OCR 和精细细节。每个样本预算：一张图像，2880 token。

多图像需要在中等分辨率（每个约 576 token）下的多张图像，以便跨图像推理能装入上下文。每个样本预算：4-8 张图像，每张 576，2300-4600 token。

视频需要在低分辨率（池化后每帧约 196 token）下的许多帧来捕捉时序动态。每个样本预算：8-32 帧，每帧 196，1600-6200 token。

如果你训练单独的模型，你选一个预算。如果你训练一个模型，你需要预算能在各场景间合理扩展而不会炸掉上下文。

在 OneVision 之前，默认答案是"训练一个场景，忽略其他"。Video-LLaVA 用额外的训练阶段将视频改造到图像模型上。LLaVA-NeXT 用切分增加了多图像支持。没有一个能干净地处理全部三种。

## 概念

### OneVision Token 预算

LLaVA-OneVision 为每个样本选择了约 3000-4000 个 token 的统一视觉 token 预算，在不同场景下分配不同：

- 单图像：AnyRes-9（3x3 瓦片 + 缩略图），每个瓦片 384 分辨率、729 个 patch，激进 2x2 双线性池化 → 每个瓦片 182。总计：9 * 182 + 182 = 1820 token。或 AnyRes-4 每瓦片 729 = 2916 + 729。
- 多图像：每张图像中等分辨率（384，无切分），729 token 无池化。预算 6 张图像 → 4374 token。
- 视频：32 帧 384 分辨率带激进 3x3 双线性池化 → 每帧 81 token。总计：32 * 81 = 2592 token。

分配维持大致恒定的总 token 数。LLM 永远不会看到超出其上下文的 batch。编码器按场景产生不同几何结构，但 LLM 消费相同预算。

### 三阶段课程

LLaVA-OneVision 分三个阶段训练：

1. 单图像 SFT（SI 阶段）。所有数据是单图像加文本。在高分辨率 AnyRes 输入上训练。这教授感知、OCR 和细粒度理解。使用 LLaVA-NeXT 数据加上 OneVision 特定单图像数据。
2. OneVision SFT（OV 阶段）。混合单图像 + 多图像 + 视频（均匀采样帧）。在统一 token 预算上训练。这教授模型处理异构 batch 形状。不从 SI 阶段重置权重——继续训练。
3. 任务迁移（TT 阶段）。以目标任务混合继续，通常多图像或视频更重，取决于产品。可选的部署微调。

关键：课程顺序很重要。先训练视频或先训练多图像会产生比先训练单图像更差的图像性能，即使数据相同。论文对此做了明确的消融。

### 为什么课程起作用

单图像训练构建感知基础。Patch token 携带细粒度视觉特征；LLM 学会将它们与文本整合。多图像和视频引入了结构性挑战（哪张图是哪个、什么先发生），没有强大的感知基础很难学习。

如果你从头一起训练所有场景，模型会感知不足（每 batch 单图像数据有限）而结构过拟合（多图像/视频数据多）。结果：一个遵循跨图像推理模式但在视觉上肤浅的模型。

课程顺序让你从 SI 阶段获得感知强度，然后从 OV 阶段获得组合/时序推理能力，不会损失任一。

### 涌现跨场景技能

LLaVA-OneVision 论文报告了三个涌现能力：

1. 多摄像头推理。分别在多图像 + 视频上训练；推理时，被要求推理一个多摄像头驾驶场景。模型正确整合了视角，尽管从未在训练中见过这种确切格式。
2. 标记集合提示。用户用编号标记注释图像中的物体；模型推理"标记 3 相对于标记 7 在做什么"。既没在标记也没在注释上训练过；从空间定位 + 多图像引用的组合中学会。
3. iPhone 截图智能体。用户提供 iPhone 屏幕截图，要求规划下一次点击。在 UI 截图、用户工作流视频和多图像前后对数据上训练。泛化到智能体用例。

这些不是训练的任务；它们从课程的组合结构中涌现出来。

### 视觉 Token 池化

Token 预算需要池化。OneVision 在 2D patch 网格上使用双线性插值：24x24 = 576 patch 变成 12x12 = 144（2 倍因子）或 8x8 = 64（3 倍因子）。池化在 patch 网格空间中完成，而非 token 空间中，以保留局部性。

每个场景的池化因子选择本身就是一个超参数。更少池化 = 更多 token = 更丰富表示。更多池化 = 更少 token = 能装下更多帧/图像。

### LLaVA-OneVision-1.5

2025 年的后续（LLaVA-OneVision-1.5, arXiv 2509.23661）在训练数据、模型权重和代码上"完全开源"。在某些基准上匹配专有模型的差距，民主化了配方。相同课程，更多数据，更好的 LLM 基础。架构无变化。

### 与 Qwen2.5-VL 的对比

Qwen2.5-VL（12.09 课）做了不同选择。它使用 M-RoPE 和动态 FPS 而非固定池化。其预算随输入变化——1 分钟视频使用的 token 比 5 秒视频多。LLaVA-OneVision 固定预算并缩放池化。两者都有效；它们以可配置性换取可预测性。

## 使用它

`code/main.py` 是一个 OneVision 风格 VLM 的课程和预算规划器。给定每个样本的 token 预算和目标场景组合（比如 40% 单图像、30% 多图像、30% 视频），它：

- 按场景分配分辨率、池化因子和帧数。
- 检查每个场景是否符合共享预算。
- 报告预期 token 数、LLM FLOPs 以及哪些场景 token 不足。
- 打印逐阶段训练调度。

用它来规划 OneVision 微调或评估 VLM 部署的每请求成本。

## 交付

本课产出 `outputs/skill-onevision-budget-planner.md`。给定目标任务分布和每个样本的预算，输出 AnyRes 因子、每帧池化、视频帧数和课程阶段权重。在训练或微调统一场景 VLM 时使用。

## 练习

1. 你的产品支持 80% 单图像、10% 多图像（2-4 张图）、10% 视频（8-16 帧）。设计 token 预算。你从不做大量多图像而省下的额外预算放在哪里？

2. 阅读 LLaVA-OneVision 第 4.3 节（涌现能力）。提出课程可能解锁但论文未报告的第四个涌现技能。

3. 交换课程顺序——先训练多图像，然后单图像，再视频。预测哪些基准会退化以及原因。

4. 论文报告仅在每个样本 8 帧上训练的视频基准。这能泛化到推理时的 30 秒视频吗？什么先崩溃——token 预算还是时序推理？

5. 24x24 patch 到 12x12 的双线性池化是每维度 4 倍缩减。用标准库 Python 实现池化，验证每个 2x2 块上的均值是否与双线性输出一致。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|---------|
| OneVision 场景 | "单图像、多图像或视频" | 统一 VLM 处理的三种输入形态之一；预算跨场景保持恒定 |
| Token 预算 | "每个样本多少 token" | LLM 每个训练/推理样本看到的视觉 token 总数，通常 3000-4000 |
| 课程 | "训练顺序" | 为涌现迁移选择的阶段顺序（单图像 → 多图像 → 视频） |
| 双线性池化 | "Token 缩减" | 在 patch 网格（2D）上应用双线性插值来减少 token 数同时保留局部性 |
| 涌现技能 | "没训练过，但仍然能行" | 推理时出现而在训练数据中没有匹配的能力，归功于课程组合 |
| AnyRes-k | "k 瓦片设置" | k 个固定分辨率子瓦片加一个缩略图，通常 k ∈ {4, 9} |
| 任务迁移 | "跨场景泛化" | 在单图像上学到的技能通过共享骨干应用于视频（反之亦然） |
