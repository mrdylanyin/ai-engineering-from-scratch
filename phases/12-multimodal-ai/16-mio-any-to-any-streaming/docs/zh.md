# MIO 与任意到任意流式多模态模型

> GPT-4o 交付了一个大多数开源模型无法复制的产品：一个能听到声音、看到视频并实时说话的智能体。开源生态到 2024 年底的答案是 MIO（Wang 等人，2024年9月）。MIO 将文本、图像、语音和音乐分词，训练一个因果 Transformer 在交错序列上，并生成任意模态到任意模态。AnyGPT（Zhan 等人，2024年2月）是概念验证；MIO 是规模化；Unified-IO 2（Allen AI，2023年12月）是带视觉+动作定位的表亲。本课阅读任意到任意模式——四种分词器，一个 Transformer，流式友好的解码。

**Type:** Learn
**Languages:** Python (stdlib, four-modality token allocator + streaming decode loop)
**Prerequisites:** Phase 12 · 11 (Chameleon), Phase 6 (Speech and Audio)
**Time:** ~120 minutes

## 学习目标

- 设计一个容纳文本、图像、语音和音乐 token 而不冲突的共享词汇。
- 比较 SEED-Tokenizer（图像）和 SpeechTokenizer 残差 VQ（语音）在压缩与重建权衡上的差异。
- 解释构建任意到任意生成的四阶段课程。
- 说出三种开源任意到任意配方及其主要权衡：MIO、AnyGPT、Unified-IO 2。

## 问题

统一多模态模型说起来容易，规模化构建难。2024年之前的大多数"任意到任意"系统是管道的：视觉模型 → 文本表示 → 语音模型 → 音频。每一跳丢失信息、增加延迟、复杂化训练。GPT-4o 的演示视频展示了一个单一模型替代方案，亚秒级响应；开源系统落后数月。

工程挑战：

- 每种模态必须存在分词器，以足够无损的压缩进行重建，并以 Transformer 能消费的速率产生 token。
- 单一词汇必须为文本（32k+）、图像（16k+）、语音（4k+）、音乐（8k+）分配空间。最低 40000+ 条目。
- 训练数据必须覆盖每个输入-输出对（文本→图像、图像→语音、语音→图像等），否则模型必须会组合。
- 推理必须足够快地流式输出 token 以满足对话延迟（<500ms 首字节音频时间）。

## 概念

### 四种模态的四种分词器

MIO 的分词器栈：

- 文本：标准 BPE，词汇 ~32000。
- 图像：SEED-Tokenizer（2023）——带离散码本的量化 VAE，4096 条目，每图像 32x32 token。
- 语音：SpeechTokenizer 残差 VQ（2023）——将 16kHz 波形编码为 8 个分层码本；第一层是粗粒度内容，后面层添加韵律和说话人身份。
- 音乐：类似残差 VQ（Meta 的 MusicGen / Encodec 家族），4-8 个码本。

每种模态产生整数 token。token 在共享词汇中获得不重叠的 ID 范围：

```
text:   0..31999
image:  32000..36095  (4096 image tokens)
speech: 36096..40191  (4096 speech base tokens, plus residual layers)
music:  40192..48383  (8192 music tokens)
sep:    48384..48390  (<image>, <speech>, <music>, </...>, etc.)
```

总计：约 48k 词汇。输入嵌入和输出投影跨越所有。

### 流式解码

语音生成使用残差 VQ。Transformer 预测基础（第 0 层）语音 token；并行解码的残差量化器预测后续层。每个第 0 层 token 在 16kHz 下约 50ms 的音频。

流式模式：

1. 用户对着麦克风说话；实时音频分词器每 50ms 发出语音 token。
2. MIO 随 token 到达消费它们（提示预填充 + 增量前向传播）。
3. 输出 token 随生成流式输出；并行语音解码器以约 50-150ms 延迟将其转换为音频采样。
4. 首字节音频时间：MIO 论文中约 300-500ms，接近 GPT-4o 的约 250ms。

Mini-Omni（arXiv:2408.16725）、GLM-4-Voice（arXiv:2412.02612）和 Moshi（arXiv:2410.00037）是互补的流式语音 LLM 设计。Moshi 特别实现了单 GPU 上 160ms 往返。

### 四阶段课程

MIO 的训练课程：

1. 阶段 1——对齐。大规模模态对语料：文本-图像、文本-语音、文本-音乐。每对使用其自己的 token 词汇段。训练共享词汇。
2. 阶段 2——交错。多模态交错文档（带图像+视频的博客、带字幕的播客等）。训练跨模态上下文。
3. 阶段 3——语音增强。额外音频数据提升语音质量而不损失文本能力。
4. 阶段 4——SFT。跨模态指令微调：VQA、标题生成、叙述、语音到语音对话。

缺少一个阶段会降低特定能力：跳过阶段 2，模型丢失跨模态上下文；跳过阶段 3，语音差。

### 视觉思维链

MIO 引入视觉思维链：模型发出中间图像 token 作为推理步骤。对于"猫在爬树吗？"模型：

1. 发出 `<image>` token 渲染场景（从输入图像或草图）。
2. 发出文本分析草图。
3. 发出最终答案。

渲染的中间图像作为草稿板。空间推理任务基准改善。这个想法镜像了文本推理的思维链。

### 任意到任意领域的竞争者

- AnyGPT（arXiv:2402.12226）：4 种模态（文本、图像、语音、音乐），类似设计。
- Unified-IO 2（arXiv:2312.17172）：增加视觉动作输出、深度、法线。更多任务多样性，更小规模。
- NExT-GPT（arXiv:2309.05519）：LLM + 模态特定扩散解码器。非单一模型方法。
- CoDi（arXiv:2305.11846）：可组合扩散；通过共享潜在空间的任意到任意。

MIO 是最接近纯 token 任意到任意的。AnyGPT 是其概念祖先。

### 延迟预算

对于对话产品，每个组件的延迟都很重要：

- 麦克风到音频 token：约 50ms。
- 预填充（音频 token + 历史）：8B 模型上约 100ms。
- 首个输出 token：约 50ms。
- 并行残差 VQ + 语音解码器：约 100-150ms。

总首字节音频时间：最小约 300ms。GPT-4o 声称约 250ms。Moshi 声称 160ms。MIO/AnyGPT 在公开基准上约 400-600ms。

### 为什么任意到任意仍然困难

即使在 2026 年，开源任意到任意模型在两个轴上落后于闭源模型：

- 语音质量。残差 VQ 分词器有损；对话语音相比 ElevenLabs 级声音听起来像机器人。
- 跨模态推理。要求模型"唱出你看到的东西"仍然比纯视觉任务更频繁失败。

这些是开放研究问题。Qwen3-Omni（12.20 课）是 2025 年最先进的开源尝试。

## 使用它

`code/main.py`：

- 定义四模态词汇分配并打印。
- 将多模态输入列表（文本、图像、音频片段、音乐）通过分词器路由器路由。
- 为文本到语音响应模拟流式解码并计延迟。
- 给定编码器、预填充和解码器延迟计算预期首字节音频时间。

## 练习

1. 你的产品接受语音输入并返回语音输出。端到端延迟预算目标是多少？列出花费时间的组件。

2. SpeechTokenizer 残差 VQ 使用 8 个码本。提出为什么并行解码残差层级是必要的（vs 顺序）以及它带来的延迟节省。

3. 你的词汇有 32k 文本 + 4k 图像 + 4k 语音。添加 8k 音乐和约 10 个分隔符。隐藏维度 4096 下嵌入矩阵的参数成本是多少？

4. 视觉思维链发出中间图像。什么类型的问题受益？什么类型被额外 token 损害？

5. 阅读 Moshi（arXiv:2410.00037）。描述其"内心独白"技术并与 MIO 的视觉思维链比较。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|---------|
| 任意到任意 | "多模态输入/输出" | 一个单一模型接受并发出文本、图像、语音和音乐，任意方向 |
| 残差 VQ | "语音分词器栈" | 多码本分词，每层增加信息；基础层是内容，后层是韵律 |
| SEED-Tokenizer | "图像编码" | MIO 使用的 4096 条目码本离散图像分词器 |
| 视觉思维链 | "视觉草稿板" | 模型生成中间图像作为最终答案前的推理步骤 |
| 首字节音频时间 | "TTFAB" | 从用户语音到首个音频输出的延迟；<500ms 为对话感 |
| 四阶段课程 | "训练配方" | 对齐 -> 交错 -> 语音增强 -> SFT，按此顺序 |
