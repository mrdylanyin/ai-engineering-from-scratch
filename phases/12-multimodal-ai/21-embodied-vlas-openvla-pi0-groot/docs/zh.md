# 具身 VLA：RT-2、OpenVLA、π0、GR00T

> 机器人第一次读取网上的食谱并在厨房机器人中执行是 RT-2（Google DeepMind，2023年7月）。RT-2 将动作离散化为文本 token，在 web 数据加机器人动作数据上共同微调一个 VLM，证明了 web 规模的视觉-语言知识可以迁移到机器人控制。OpenVLA（2024年6月）发布了开源 7B 参考。Physical Intelligence 的 π0 系列（2024-2025）添加了流匹配动作专家。NVIDIA 的 GR00T N1（2025年3月）为人形机器人大规模提供了双系统（系统1/系统2）控制。VLA 原语——视觉-语言-动作，一个单一模型看到、阅读并行动——是本阶段理解模型与 Phase 15 自主系统之间的桥梁。

**Type:** Learn
**Languages:** Python (stdlib, action tokenizer + VLA inference skeleton)
**Prerequisites:** Phase 12 · 05 (LLaVA), Phase 15 (Autonomous Systems, referenced)
**Time:** ~180 minutes

## 学习目标

- 描述动作分词：离散区间编码（RT-2）、FAST 高效动作 token、连续流匹配动作（π0）。
- 解释为什么在 web + 机器人数据上共同微调保留了通用知识向新任务的迁移。
- 在同一机器人任务上比较 OpenVLA（开源 7B Llama+VLM）、π0（流匹配）和 GR00T N1（双系统）。
- 说出 Open X-Embodiment 数据集及其作为 RT-X 训练语料的角色。

## 问题

一个从自然语言指令中做家务的机器人，自 20 世纪 70 年代以来一直是研究目标。2020 年代的答案：视觉-语言-动作（VLA）模型。与 VQA 使用的相同 VLM 架构，但输出是动作（关节扭矩、末端执行器位姿、离散命令）而非文本。

VLA 特有的挑战：

1. 动作空间是连续的（关节角度、力）和高维的（7-DOF 手臂 + 3-DOF 夹爪 = 10 维，在 30 Hz 下）。
2. 机器人特定训练数据稀缺。Open X-Embodiment 约 100 万轨迹；web 文本-图像是 50 亿+。
3. 控制频率很重要。30 Hz 控制循环意味着每动作 33ms 预算。
4. 安全性。错误动作会损坏硬件、伤害人类或造成财产损失。

## 概念

### 动作分词（RT-2）

RT-2 的技巧：将每个关节目标表示为量化的文本 token。将归一化的 [-1, 1] 范围离散化为 256 个区间，每个区间映射到一个词汇 ID。一个 10-DOF 动作在每个控制步骤变成 10 个 token。

在混合上共同微调 PaLM-X VLM：

- Web 图像-文本对（标题生成、VQA）。
- 机器人演示，动作作为 token。

模型看到 "pick up the red cube"（语言）→ 图像（视觉）→ 10 token 动作序列（离散化关节目标）。Web 预训练保留通用知识迁移：RT-2 可以跟随 "move towards the fast-moving object"，即使 "fast-moving" 不在训练数据中。

RT-2 论文中推理在 3-5 Hz，受限于 VLM 自回归解码。

### OpenVLA——开源 7B 参考

OpenVLA（Kim 等人，2024年6月）是开源权重的 RT-2 等效。7B Llama 骨干，DINOv2 + SigLIP 双视觉编码器，256 区间动作分词。

在 Open X-Embodiment（22 个机器人、97 万轨迹）上训练。提供 LoRA 微调支持以适配新机器人。

推理：在 A100 上带量化 4-5 Hz。对慢速操作足够，对高频控制不够。

### FAST 分词器——更快的动作解码

Pertsch 等人（2024）展示了离散区间分词是低效的——大多数动作聚集在区间空间的小区域。FAST（频率域动作序列分词器）通过 DCT 压缩动作序列并量化系数。

一个 30 步动作轨迹变成约 10 个 FAST token 而非 300 个离散区间 token。推理快 3-5 倍而无质量损失。

### π0 和流匹配动作

Physical Intelligence 的 π0（Black 等人，2024年10月）用流匹配动作专家替换了离散动作 token：

- 一个小子动作 Transformer 读取 VLM 的隐藏状态，通过整流流输出连续 50 步动作序列。
- 动作头用流匹配损失训练；VLM 预训练保持不变。
- 推理：完整动作序列在约 5 个去噪步骤中发出，实际达到 50 Hz 控制。

π0 的声明：在广泛的操作任务套件上击败 OpenVLA 和 Octo。连续动作公式保留了离散化所破坏的平滑性。

π0.5 和 π0-FAST 是增量升级。π0-FAST 结合了 FAST 分词和流匹配。

### GR00T N1——人形机器人双系统

NVIDIA 的 GR00T N1（2025年3月）为人形机器人（>30 DOF，全身）构建：

- 系统 2：大型 VLM 读取场景 + 指令，以约 1 Hz 产生高层子目标。
- 系统 1：小子动作头 Transformer，在子目标条件化下以 50-100 Hz 产生低层关节命令。

这种分离映射到 Kahneman 的快慢思考：系统 2 规划，系统 1 执行。好处：慢 VLM 规模规划不阻塞快速控制；系统 1 保持小以保证延迟。

GR00T N1.7（2025年末）改进了数据扩展。GR00T 用来自 Omniverse 的仿真到真实数据做微调。

### Open X-Embodiment

训练数据。RT-X（2023年10月）整合了 22 个数据集，覆盖 100 万轨迹，横跨 22 个机器人。Open X-Embodiment 是所有人使用的语料：

- ALOHA / Bridge V2 / Droid / RT-2 Kitchen / Language Table。
- 每个样本：（机器人状态、摄像头视角、指令、动作序列）。
- 训练卫生：统一动作空间、归一化关节范围、调整摄像头尺寸。

OpenVLA 和 π0 在 Open X-Embodiment 上训练。到任何特定机器人的领域差距通过在 100-1000 个任务特定演示上做 LoRA 微调来弥合。

### 共同微调 vs 仅机器人

共同微调混合了 web VQA 数据和机器人轨迹。比例很重要：太多 VQA，模型忘记动作；太多机器人数据，模型失去通用知识。

RT-2 的比例：约 1:1。OpenVLA：约 0.5:1 web 对机器人。π0：类似。精确比例是一个随数据集大小调优的超参数。

仅机器人训练产生对分布外指令失败的任务特定模型。共同微调是"捡起红色方块（演示中听过）"和"捡起从左数第三大的物体（新颖措辞）"之间的区别。

### 安全与动作限制

每个生产 VLA 都附带：

- 硬关节限制（不能超出规格扭矩）。
- 速度限制（软裁剪）。
- 工作空间边界（末端执行器不能离开桌面）。
- 新颖任务的人类在环审批。

这些作为控制层检查坐落于 VLA 之外。VLA 的输出是建议，不是命令。

## 使用它

`code/main.py`：

- 实现 256 区间动作分词和反分词。
- 基于 DCT + 量化的 FAST 分词器草图。
- 跨（离散区间、FAST、连续流）比较每动作步骤的 token 数。
- 打印 RT-2 → OpenVLA → π0 → GR00T 谱系摘要。

## 交付

本课产出 `outputs/skill-vla-action-format-picker.md`。给定机器人任务（操作、导航、人形全身），在离散区间 + RT-2、FAST + OpenVLA、流匹配 + π0 或双系统 + GR00T 之间选择。

## 练习

1. 10-DOF 手臂在 30 Hz 控制率下。256 区间离散区间分词每秒发出多少个 token？7B VLM 能跟上吗？

2. FAST 分词将 30 步轨迹压缩到约 10 个 token。如果轨迹有高频运动（如打鼓），用户会失去什么？

3. π0 的流匹配头约 5 步去噪。比较吞吐量与 OpenVLA 在 4-5 Hz 的自回归解码。

4. GR00T 的系统1/系统2 分离映射到 Kahneman。提出一个可能帮助双足行走的不同分离方案（系统3？）。

5. 阅读 Open X-Embodiment 第 4 节关于数据集策划。说出防止领域泄漏的三条策划规则。

## 关键术语

| 术语 | 人们怎么说 | 实际含义 |
|------|-----------|---------|
| VLA | "视觉-语言-动作" | 接受图像 + 指令并输出动作命令的模型 |
| 动作分词 | "离散区间" | 将连续关节目标每维量化为 256 个区间，每个是词汇 ID |
| FAST 分词器 | "频率动作 token" | DCT + 量化将 30 步轨迹压缩到约 10 个 token |
| 共同微调 | "混合 web + 机器人" | 在 web VQA 数据与机器人演示一起训练以保留通用知识 |
| 流匹配动作头 | "π0 连续输出" | 通过整流流输出 50 步动作序列的小子 Transformer |
| 系统1/系统2 | "双系统控制" | 大型 VLM 慢规划，小子动作头快执行；GR00T 模式 |
| Open X-Embodiment | "RT-X 数据集" | 100 万轨迹的跨机器人数据集；训练语料 |
